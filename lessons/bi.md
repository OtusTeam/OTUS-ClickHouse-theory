## Введение в Business Intelligence (BI)

### Что такое BI?
BI (Business Intelligence) — это эволюционирующая концепция поддержки принятия решений, которая включает в себя несколько аспектов:
- Это информационная система.
- Это процесс анализа данных.
- Это система принятия решений.

**Эволюция концепции:**
- **Тогда:** Информационная система руководителя (EIS).
- **Сейчас:** Информационная система каждого (BI).

### Определение BI
- **BI** — это термин, объединяющий **архитектуры, инструменты, базы данных, аналитические средства, приложения и методологии**.
- **Основная задача:** Обеспечить легкий доступ к данным (и моделям), чтобы предоставить бизнес-менеджерам возможность проводить анализ.
- **Конечная цель:** Помогать преобразовывать данные в информацию (и знания), затем в решения и, наконец, в действия.

### Краткая история BI
Термин был введен Gartner в середине 1990-х, но концепция старше:
- **1970-е:** MIS (Management Information Systems) — статические/периодические отчеты.
- **1980-е:** EIS (Executive Information Systems) — информационные системы руководителей.
- **1990-е:** Появление OLAP, динамических и многомерных отчетов, ad-hoc отчетности. Возникновение термина "BI".
- **2005+:** Интеграция с AI, поиск по данным/тексту, веб-порталы и приборные панели (dashboards).

### Ключевые идеи и функции BI
- **Основные идеи:**
    - BI — это методы и инструменты для бизнес-анализа.
    - BI упрощает доступ к данным в Хранилище Данных (DWH). **Без DWH нет BI.**
    - SQL по-прежнему более чем актуален.
- **Основные функции (фичи):**
    - Визуальный конструктор запросов.
    - Предопределенные метрики, запросы и отчеты.
    - Формирование информационных панелей (дашбордов).
    - Уведомления, алерты и рассылки по расписанию.
    - Гибкие возможности публикации (sharing).

### Требования к BI-инструментам
При выборе инструмента стоит обращать внимание на:
1.  Поддержка необходимых коннекторов и функционала.
2.  Незатратность настройки.
3.  Надежность и отказоустойчивость.
4.  Возможность мониторинга.
5.  Визуальная составляющая.
6.  Комьюнити и распространенность.
7.  Требования бизнес-пользователей.

## Классификация BI-систем

### По физическому нахождению данных
1.  **Облачные (SaaS, Cloud-based):** Данные находятся в облаке.
2.  **В собственном хранилище (Self-hosted):** Для компаний, где недопустима утечка информации. Развертывается на своих серверах.
3.  **На арендованном сервере (Vendor-hosted):** Для клиентов, которые не хотят содержать инженера и оборудование.

### В зависимости от стоимости
1.  **Бесплатные (Open-Source):** Примеры: *Yandex Datalens, Apache Superset*.
2.  **Условно-бесплатные:** Стоимость зависит от типа использования. Пример: *QlikView*.
3.  **Платные:** Примеры: *Klipfolio, Looker*.

## Apache Superset

### Об инструменте
- **Explore:** Apache Superset предоставляет возможность исследовать данные.
- **Поддерживаемые источники:** Superset работает с большим количеством баз данных, включая ClickHouse, а также "many other compatible databases".
- **Сообщество:** У Apache Superset есть активное сообщество.
    - **Русскоязычное сообщество:** Telegram: t.me/superset_ru
    - **Участие:** Slack, GitHub, Stack Overflow, встречи и гайды для контрибьюторов.

### Практическая работа с Superset (Возможности)
В ходе практики рассматриваются следующие возможности:
- Создание наборов данных (datasets).
- Создание визуализаций (visualizations).
- Создание графиков (charts).
- Создание информационных панелей (dashboard).
- Настройка оповещений и мониторинга (alerting/monitoring).
- Работа в SQL Lab.
- Управление пользователями, ролями и т.д.
